한 줄 요약

하나의 TCP 위에서
양방향으로 주고받는다.
전이중(Full-Duplex) 통신 프로토콜 — 서버가 먼저 보낼 수 있다.
##

왜 WebSocket인가?

HTTP는 요청해야
응답받는다.
새 데이터가 있는지 매번 물어봐야 한다. 채팅·시세·게임엔 비효율적.
##

HTTP 실시간 시도들

①Polling주기적 반복 요청 · 불필요한 요청 폭증 · 지연
②Long Polling데이터 있을 때까지 응답 보류 · 연결 점유 · 재연결 비용
③SSE서버→클라 단방향 스트림 · 단방향 · 바이너리 미지원
WebSocket은 이 문제들을 근본적으로 해결한다.
##

핸드셰이크 — HTTP Upgrade

101
Client → Server (HTTP GET)
GET /chat HTTP/1.1
Upgrade: websocket
Connection: Upgrade
Sec-WebSocket-Key: dGhlIHNhbXBsZSBub25jZQ==
서버는 101 Switching Protocols로 응답 → 이후 HTTP는 더 이상 쓰이지 않는다.
##

Accept 키 검증

magic string과 SHA-1
Sec-WebSocket-Accept =
  Base64(SHA-1(Key + "258EAFA5-E914-47DA-95CA-C5AB0DC85B11"))
🔑
이건 보안이 아니라 프로토콜 확인이다 잘못된 HTTP 응답이 WebSocket으로 오해석되는 것을 방지한다.
##

프레임 — opcode

0x1TextUTF-8 텍스트 메시지
0x2Binary바이너리 데이터
0x8Close연결 종료 프레임
0x9Ping생존 확인 요청
0xAPongPing에 대한 응답
!!

마스킹 — 클라→서버 필수

🛡️
클라이언트 → 서버 프레임은 반드시 마스킹(MASK=1). 서버 → 클라이언트는 0. 캐시 오염(Cache Poisoning) 방지 — 중간 프록시가 프레임을 HTTP 응답으로 오해석해 캐시에 저장하는 것을 막는다.
Payload len 7bit: 0~125 / 126→2byte / 127→8byte로 확장.

##

Heartbeat — Ping/Pong

WebSocketConfig.java (STOMP)
config.enableSimpleBroker("/topic", "/queue")
      .setHeartbeatValue(new long[]{10000, 10000}); // 10s
NAT·프록시·방화벽이 유휴 연결을 끊는다. Ping/Pong으로 생존 확인 + 좀비 연결 정리.
##

재연결 — Backoff + Jitter

WebSocketClient.js
const jitter = Math.random() * 1000;
const delay = Math.min(base * 2**attempts, 30000) + jitter;
// 비정상 종료(code !== 1000)만 재연결
⚡
Jitter가 없으면 Thundering Herd 서버 재시작 시 모든 클라이언트가 동시에 재연결 → 주기적 스파이크로 복구 불가.

STOMP · SockJS

WebSocket은 메시지 형식을 정의하지 않는다. STOMP가 Pub/Sub·라우팅을 얹고, SockJS가 미지원 브라우저 폴백.
!!

Scale-Out의 함정

서버가 늘어나면…
// A는 Server 1, C는 Server 2에 연결
A ──ws──▶ Server 1   (Session: A, B)
C ──ws──▶ Server 2   (Session: C, D)

A → C 메시지?  // Server 1엔 C의 세션이 없다!
연결이 특정 서버에 묶이는 Stateful 특성 때문에 서버 간 전달이 안 된다.
##

해결 — Pub/Sub 백플레인

Redis Pub/Sub로 서버 간 브로드캐스트
A ──▶ Server 1 ──pub──▶ Redis Pub/Sub ──sub──▶ Server 2 ──▶ C
// 대안: Kafka / RabbitMQ

# Nginx Sticky Session
upstream ws { ip_hash; }   # 같은 IP → 같은 서버
정석: Sticky Session(ip_hash) + Redis Pub/Sub 백플레인. proxy_http_version 1.1 필수.

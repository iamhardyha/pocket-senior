# Note Authoring — Pocket Senior 노트 규약

면접 대비 백엔드 미니북의 노트는 **밀도 높은 읽기 자료**다. 한 편이 한 질문을 깊게 답한다.

## 노트 포맷 (본문 구조)

```markdown
# {제목}

> **핵심 질문**: {사용자의 질문}

---

## 한 줄 요약
**"{핵심을 한 문장으로}"**

---

## {본문 섹션들}
(깊이 있는 설명, 코드 예시, 테이블, 다이어그램. 왜·트레이드오프·실무 함정 포함)

---

## 면접에서 이렇게 답하자
(실전 답변 가이드)
```

**Why 이 구조인가**: 한 줄 요약은 출퇴근 중 빠른 복습용, 본문은 깊은 이해용, 면접 가이드는 실전 전환용 — 세 층이 서로를 보완한다.

## Frontmatter (필수)

```yaml
---
tags: [영문기술키워드, 한국어개념키워드, ...]   # 3~6개. 예: [Redis, Kafka, 캐싱, 장애복구]
question: "질문목록·갤러리에 표시될 질문 텍스트"
status: 🟢                                      # 🔴 미학습 / 🟡 학습중 / 🟢 완료
order: {카테고리 내 마지막 order + 1}            # 정수
slides: true                                     # 슬라이드 덱 생성 후 slide-deck-builder가 추가 (작성 시점엔 생략 가능)
---
```

- `tags`: 기술 키워드(영문)+개념 키워드(한국어) 혼합 3~6개.
- `order`: 같은 카테고리의 기존 최대 order를 확인하고 +1.
- `slides`는 덱이 생성된 뒤에 켠다 — 노트만 있고 덱이 없는데 켜면 깨진 링크가 된다(고아 토글 방지).

## 카테고리 & 저장 경로

| 카테고리 | 디렉토리 | 도메인 |
|---|---|---|
| traffic | `docs/traffic/` | 트래픽 & 장애 대응 |
| concurrency | `docs/concurrency/` | 데이터 정합성 & 동시성 |
| failure | `docs/failure/` | 장애 시나리오 |
| database | `docs/database/` | 데이터베이스 |
| architecture | `docs/architecture/` | 아키텍처 & 비동기 |
| infra | `docs/infra/` | 인프라 & 마이그레이션 |

- 저장: `docs/<category>/<제목>.md`
- 새 카테고리가 필요하면 `docs/<new-category>/` 생성 (드문 경우 — 먼저 기존에 안 맞는지 확인).
- **파일명**: 한국어 kebab-case (예: `락과-동시성-제어.md`). 갤러리·덱 경로가 이 파일명을 그대로 쓴다.

## 내부 링크
- 같은 디렉토리: `./file.md`
- 다른 디렉토리: `../category/file.md`

## 품질 체크
- 사용자가 막혔던 지점이 본문에서 실제로 풀리는가 (스코프 충실도)
- 단정적 수치/"항상·반드시"에 근거가 있는가 — 없으면 "상황에 따라"
- 자신 없는 주장은 ❓로 표시해 검증가가 집중하게 한다
